## 회의 · 협업 규칙

| 분류 | 항목 | 내용 |
|---|---|---|
| **회의** | 정기 회의 | 매주 월요일 19:00 · Google Meet · 30분 |
| | 회의록 | `docs/회의록_N차.md` 형식 · 팀장 주도 |
| | 진행 상태 공유 | WBS 작업 상태 업데이트 후 회의 |
| | 긴급 상황 | Slack/카톡방 즉시 공유 · 24시간 내 응답 |
| **Git** | 브랜치 전략 | `main` (보호) + `feature/기능명` · 완료 시 PR → 리뷰 → merge |
| | 커밋 메시지 | `feat:` 기능 · `fix:` 버그 · `docs:` 문서 · `refactor:` 리팩토링 (Conventional Commits) |
| | PR 규칙 | 최소 1명 리뷰 필수 · 본인 머지 금지 · `[skip ci]` 금지 |
| | 충돌 해결 | pull 먼저 · 로컬에서 해결 후 push · 불확실하면 팀원과 상의 |
| **코드** | 네이밍 규칙 | Java: `camelCase` · 클래스 `PascalCase` · 상수 `UPPER_SNAKE` |
| | 한국어 주석 | 핵심 로직에만 간결하게 · 코드로 설명 가능하면 주석 불필요 |
| | 코드 리뷰 | Must 기능 PR은 반드시 팀원 리뷰 |
| **문서** | `docs/` 구조 | `W08_요구사항_정의서.md` · `W09_ERD.md` · `W10_화면설계서.md` · `W10_API_명세서.md` · `W11_WBS.md` · `회의록_N차.md` |
| | 커밋 접두사 | `docs:` · 산출물 변경 이력 추적 |
| | 발표자료 | `docs/presentation/` 폴더에 W14 · W15 자료 보관 |
